## Как запустить
1. `git clone <URL_вашего_репозитория>`
2. `cd todo_app && dart pub get`
3. `dart run bin/todo_app.dart`

## Что изучили
- Основы Dart: Sound Null Safety, вывод типов (`var`), `final` vs `const`.
- Работу с коллекциями, функциями, именованными параметрами и классами.
- Создание CLI-приложения с `dart:io` и обработкой ввода.
- Подключение пакетов через `pub.dev` (`ansicolor`).
- Асинхронность: `Future`, `async/await`, неблокирующий event loop.

## Ответы на вопросы
1. **Чем `final` отличается от `const`?**  
   `final` вычисляется в runtime и инициализируется один раз. `const` вычисляется строго на этапе компиляции, требует неизменяемого значения.
2. **Что означает `String?`?**  
   Это nullable-тип. Переменная может содержать строку или `null`. Dart требует явной обработки `null` благодаря Sound Null Safety.
3. **Чем `Future` отличается от обычного значения? Что означает `await`?**  
   `Future` – это обещание результата, который появится позже. `await` приостанавливает выполнение текущей функции, не блокируя поток, и передает управление event loop.
4. **Зачем в Dart именованные конструкторы?**  
   В Dart нет перегрузки методов. Именованные конструкторы (`ClassName.name()`) позволяют создавать разные варианты инициализации с уникальными сигнатурами, улучшая читаемость.
